# Public JSON Contract

This project exports static JSON read models under `data/public/` for the frontend.

All files are generated by [`build_graph.py`](../apps/pipeline/build_graph.py) and validated before write.

## Core entity exports

### `jobs.json`

Array of normalized jobs for list/detail views.

Key fields:

- `id`
- `company_id`
- `company_name`
- `title`
- `location_text`
- `remote_mode`
- `employment_type`
- `posted_at`
- `source_type`
- `source_url`
- `description_hash`
- `dedupe_fingerprint`
- `status`
- `metadata`
- `enrichment`

### `companies.json`

Array of company entities.

Key fields:

- `id`
- `name`
- `aliases`
- `website`
- `industry`
- `metadata`

### `skills.json`

Array of skill entities.

Key fields:

- `id`
- `label`
- `normalized_label`
- `aliases`
- `parent_id`
- `category`
- `metadata`

## Graph export

### `graph.full.json`

Object with:

- `nodes`: array of graph nodes
- `edges`: array of graph edges

Node shape:

- `id`
- `type`
- `label`
- `data`

Edge shape:

- `id`
- `source`
- `target`
- `type`
- `weight`
- `confidence`
- `data`

## Read models

### `company_jobs.json`

Object map:

- key: `company_id`
- value: array of `job_id`

### `skill_jobs.json`

Object map:

- key: `skill_id`
- value: array of `job_id`

### `job_skills.json`

Object map:

- key: `job_id`
- value: array of objects with:
  - `skill_id`
  - `label`
  - `edge_type`
  - `weight`
  - `confidence`

### `company_skill_stats.json`

Object map:

- key: `company_id`
- value: object map of:
  - key: `skill_id`
  - value: integer count

### `job_neighbors.json`

Object map:

- key: `job_id`
- value: object with:
  - `similar_jobs`
  - `skills`
  - `company_id`

`similar_jobs` items:

- `job_id`
- `score`

`skills` items follow the `job_skills.json` item shape.

### `search-index.json`

Array of lightweight search docs.

Each item includes:

- `id`
- `type`
- `title`
- `text`

### `summary.json`

Top-level generation summary.

Fields:

- `generated_at`
- `counts`

`counts` includes:

- `companies`
- `jobs`
- `skills`
- `nodes`
- `edges`

## Stability notes

- These files are the frontend contract and should change deliberately.
- If a file shape changes, update the matching schema in [`packages/schema`](../packages/schema).
- `summary.json` and the read-model maps are intended for fast UI reads, not for lossless reconstruction of the SQLite truth layer.
